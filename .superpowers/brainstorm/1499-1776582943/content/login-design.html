Key details

  • Login form replaces the todo list area — header stays visible
  • "Create one" toggles the form to a sign-up mode (same card, extra confirm-password field)
  • Auth errors shown inline below the submit button (wrong password, email taken, etc.)
  • When logged in: user's email + "Sign out" button appear in the header next to the gear icon
  • Todos are scoped to the logged-in user via userId (already supported by todosService)
